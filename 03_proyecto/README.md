# 🎬 Proyecto: Análisis de Datos de Netflix (Día 3)

## 📝 Descripción del Proyecto
Este proyecto fue desarrollado como parte práctica del curso **SQL TOTAL**. El objetivo principal fue adoptar el rol de un analista de datos para Netflix para extraer *insights* clave a partir de su base de datos. Este análisis permite una mejor comprensión de la distribución de su contenido, la duración de sus episodios y el impacto de su talento en la plataforma.

## 🎯 Objetivo
Analizar la estructura de datos relacional de la base de datos `netflixdb` para responder preguntas de negocio críticas que faciliten la toma de decisiones estratégicas sobre el catálogo.

---

## 🗄️ Estructura de la Base de Datos
El análisis se centró en las siguientes entidades clave:
*   **`series`**: Información general de las series (títulos, años de lanzamiento, rating promedio).
*   **`actores`**: Datos del talento y su participación en las producciones.
*   **`episodios`**: Detalle de cada capítulo, incluyendo su duración y métricas individuales.

## 🚀 Preguntas de Negocio Resueltas

Para este análisis, se formularon y resolvieron las siguientes consultas estratégicas utilizando SQL:

1. **Actor/Actriz con mayor participación:** ¿Quién ha aparecido en el mayor número de series?
2. **Calidad de contenido:** ¿Cuál es la serie con el mejor rating promedio según IMDB?
3. **Duración de contenido:** ¿Cuál es el episodio con la mayor duración (el más largo)?
